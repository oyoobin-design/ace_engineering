## 1. 뉴스 수집 & 필터링 로직

### 전체 흐름

```
① 파싱
   RSS 13개 피드 → 기사 제목·링크 추출
   (매경·한경·연합뉴스·전자신문, 24시간 이내만)
         ↓
② 임베딩 = 벡터 변환 (같은 단계)
   기사 제목들 → OpenAI API → 숫자 목록으로 변환
   에이스 맥락도 → 숫자 목록으로 변환
   두 벡터 간 거리 계산 → 가까운 상위 80개만 남김
         ↓
③ GPT 배치 선별
   80개를 30개씩 나눠서 GPT에 동시 전송
   GPT가 맥락 이해해서 진짜 관련 있는 것만 + 이유 붙여서 반환
```

---

### ① 파싱이란?

RSS = 언론사가 최신 기사를 자동으로 제공하는 피드. XML 형식.

```
매일경제 RSS → <item><title>EU 탄소 규제...</title><link>https://...</link></item>
```

코드가 이 XML을 읽어서 제목·링크·날짜를 뽑아냄.
24시간 이내 기사만 통과. 유료 기사(plus.hankyung.com 등) 자동 제외.

**언론사별 수집 섹션:**

| 언론사 | 섹션 |
|--------|------|
| 매일경제 | 경제·기업경영·국제 |
| 한국경제 | 경제·IT·국제 |
| 연합뉴스 | 경제·산업·세계 |
| 전자신문 | 전자·AI·정책·국제 |

---

### ② 임베딩(벡터 변환)이란?

**임베딩 = 텍스트를 숫자 목록(벡터)으로 변환하는 것.**
임베딩과 벡터 변환은 같은 말.

```
"EU 탄소 규제 강화"  → [0.23, 0.87, 0.11, ... 1536개 숫자]
"에이스엔지니어링"   → [0.21, 0.85, 0.13, ... 1536개 숫자]  ← 비슷함
"농협은행 AI 인수"   → [0.91, 0.02, 0.67, ... 1536개 숫자]  ← 다름
```

숫자가 비슷하면 = 의미가 비슷하다.

**벡터란?** 그냥 숫자 목록. 지도 좌표(위도, 경도)처럼 위치를 나타내는데,
여기선 '의미의 위치'를 1536개 숫자로 표현하는 것.

**코사인 유사도란?** 두 벡터가 얼마나 같은 방향을 가리키는지 측정.
1에 가까울수록 유사, 0에 가까울수록 다름.

**이 단계에서 하는 일:**
1. 에이스엔지니어링 맥락을 벡터로 변환 (1번만)
2. 수집된 기사 제목 전체를 벡터로 변환 (한 번에 배치 처리)
3. 에이스 벡터와 각 기사 벡터의 유사도 계산
4. 유사도 높은 상위 80개만 다음 단계로 전달

**왜 키워드 필터 안 씀?**
키워드는 단어가 있어야만 잡음. 임베딩은 단어 없어도 의미가 비슷하면 잡음.
예: "구글 야간 전력 조달 의무화" → ESS·배터리 키워드 없어도 에이스랑 관련 있다고 판단 가능.

**사용 모델:** `text-embedding-3-small` (OpenAI)
**비용:** 기사 500개 변환해도 $0.0002 수준 (거의 공짜)

---

### ③ GPT 배치 처리란?

임베딩으로 추린 80개를 GPT에게 보내서 최종 판단.

**배치 처리가 필요한 이유:**
80개를 한 번에 보내면 → GPT가 느려지거나 타임아웃.
30개씩 나눠서 동시에 보내면 → 3묶음이 병렬로 처리되어 빠름.

```
80개
  ↓ 30개씩 자르기
묶음1 (0~29번)  ─┐
묶음2 (30~59번) ─┤ 동시 전송 → ~5초
묶음3 (60~79번) ─┘
  ↓
결과 합치기 → 최종 관련 기사 + 이유
```

**배치 수는 자동 계산:**
```
배치 수 = 올림(기사 수 ÷ 30)
기사 45개 → 2배치 / 기사 90개 → 3배치 / 기사 12개 → 1배치
```

**GPT가 하는 일:**
- 각 기사 제목이 에이스엔지니어링과 관련 있는지 맥락으로 판단
- 관련 있으면 번호 + 이유("IRA 변동 → 에이스 미국 수주 영향") 반환
- 관련 없으면 제외

**사용 모델:** `gpt-4o`
**단독 기사 우선:** 제목에 "단독" 포함 시 임베딩 점수 +0.05 보너스

## 2. 병렬 처리 로직

### RSS 수집 병렬화

```
매경 경제    ─┐
매경 기업경영 ─┤
매경 국제    ─┤
한경 경제    ─┤
한경 IT     ─┤  Promise.all (13개 동시 요청)
한경 국제    ─┤  → 가장 느린 피드 1개 응답시간만 기다림
연합 경제    ─┤
연합 산업    ─┤
연합 세계    ─┤
전자 전자    ─┤
전자 AI     ─┤
전자 정책   ─┤
전자 국제   ─┘
```

순차 처리였으면 13번 기다려야 하지만, 병렬이라 동시에 요청.

### GPT 배치 병렬화

30개씩 나눈 배치도 `Promise.all`로 동시 전송.
순차 처리(배치1 끝나면 배치2) 대신 모두 동시에 시작.

## 3. EIA API 로직

**EIA = U.S. Energy Information Administration (미국 에너지부 산하 통계청)**

미국 내 배터리 ESS 설치 용량을 연도별로 제공하는 공식 무료 API.

```
GET https://api.eia.gov/v2/electricity/...
  → { period: "2024", value: 16800, unit: "MW" }
```

- API 키 없으면 → 데모 데이터 자동 반환 (차트는 보임)
- 캐시: 24시간 (하루 1번만 실제 API 호출)

## 4. PPT 자동생성 로직

```
① 클라이언트(브라우저)
   html2canvas로 차트 이미지 캡처 (EIA 차트, 경쟁사 맵)
         ↓
② 서버 API (/api/report)
   EIA 데이터 + 뉴스 → GPT-4o → 슬라이드 텍스트 생성
         ↓
③ 서버 API (/api/pptx)
   텍스트 + 차트 이미지 → pptxgenjs → .pptx 파일 생성
   (pptxgenjs는 Node.js 전용이라 반드시 서버에서 실행)
         ↓
④ 클라이언트
   .pptx 파일 다운로드
```

**왜 PPT 생성을 서버에서 하나?**
`pptxgenjs` 라이브러리가 `node:fs`(파일 시스템)를 사용하는데,
브라우저에는 파일 시스템이 없어서 클라이언트에서 실행 불가.
→ Next.js API 라우트(서버)에서 실행 후 파일로 전송.

## 5. 경쟁사 뉴스 로직

경쟁사 탭은 한국 언론 RSS 대신 **Google News RSS 직접 검색** 사용.
이유: 한국 언론에 Rittal·TLS Energy 같은 해외 기업 기사가 거의 없음.

```
Rittal 클릭
  → /api/competitor-news?competitor=rittal
  → Google News RSS 검색: "Rittal battery energy storage enclosure"
  → 24시간 이내 기사 최대 5개 반환
```

**경쟁사별 검색어:**

| 경쟁사 | 검색어 |
|--------|--------|
| TLS Energy | TLS Energy enclosure battery storage |
| Rittal | Rittal battery energy storage enclosure |
| 신성에스티 | 신성에스티 ESS 에너지저장 배터리 |
| CATL | CATL enclosure battery storage ESS 2025 |
| Fluence | Fluence Energy FLNC ESS project |

## 6. 자주 받을 수 있는 질문 & 답변

**Q. 왜 키워드 필터 안 쓰고 임베딩을 쓰나요?**
키워드는 단어가 정확히 일치해야만 작동합니다. 하지만 "구글이 야간 전력 확보를 위해 ESS 투자 확대"처럼 ESS 키워드가 없어도 에이스엔지니어링에 직결되는 기사가 있습니다. 임베딩은 의미로 판단하기 때문에 이런 기사도 잡을 수 있습니다.

**Q. 임베딩이랑 GPT를 둘 다 쓰는 이유가 뭔가요?**
임베딩은 빠르고 저렴하지만 "왜 관련 있는지" 설명 못 합니다. GPT는 느리고 비싸지만 맥락 이해와 이유 설명이 됩니다. 임베딩으로 후보를 좁히고 GPT로 최종 판단 — 속도와 정확도를 동시에 잡는 구조입니다.

**Q. 뉴스가 실시간인가요?**
언론사 RSS를 30분 캐시로 수집합니다. 완전 실시간은 아니지만 최대 30분 지연으로 최신 기사를 반영합니다.

**Q. 비용이 얼마나 드나요?**
- 임베딩: 기사 300개 처리 시 약 $0.0003 (한 달 매일 써도 $0.01)
- GPT 배치: 배치 3개 × 호출당 약 $0.01 = $0.03/회
- 하루 10회 사용 기준 월 $9 수준
