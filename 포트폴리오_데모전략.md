# 에이스엔지니어링 사업기획 — 포트폴리오 데모전략 (JD 정합 v2)

> 목적: **AX 능력 + 빠른 적응력 + 적극성** 증명
> 핵심 원칙: 공고(JD)가 원하는 것에 도구를 정확히 맞춘다. 핵심은 완벽히 돌리고(데모), 야심은 로드맵으로 설명한다(비전).
> 관련 문서: 데이터·API 상세는 `JD_도구_API_분석.md`, 보고서 출력 예시는 `에이스_PPT자동생성_샘플.pptx`.

## 0. 한 줄 정체성 (영업 아님, 기획임)

> **"ESS·데이터센터 시장분석 + 파트너 리스크 모니터링 + 사업계획·보고 자동화"**
> — 개별 바이어를 발굴·클로징하는 *영업*이 아니라, 어느 시장/세그먼트에 베팅하고 어디에 투자할지를 데이터로 좁혀주는 *사업기획*.

| | 해외영업 (이 도구 아님) | 사업기획 (이 도구) |
|---|---|---|
| 질문 | "이 바이어 언제 RFQ, 얼마 부를까" | "어느 시장이 크고 우리는 어디에 베팅하나" |
| 단위 | 개별 딜 | 시장·세그먼트·포트폴리오 |
| 산출물 | 수주 계약 | 투자·생산능력·신사업 의사결정 근거 |

NDA 문제는 자동 해소: 에이스가 **홈페이지에 파트너사를 공개**(Fluence·Powin·삼성SDI·LG엔솔·CATL 등 38곳)하므로, 개별 바이어가 아니라 *공개 파트너의 시장 활동*을 모니터링하는 구조라 기밀과 무관.

## 1. JD 정합성 — 면접의 핵심 카드

공고 담당업무에 도구를 1:1로 맞췄다는 것을 보여주는 표. *"공고를 보고 이렇게 맞췄습니다"*가 적극성의 증거.

| 공고 담당업무 | 도구(탭) | 상태 |
|---|---|---|
| 시장분석 (ESS) | 시장분석 — EIA + 글로벌·교역 | 🟢 |
| 시장분석 (**Data Center**) | 데이터센터 탭 (신규시장) | 🔴 신규 |
| 시장분석 (신규시장) | 국가비교 (World Bank) | 🔴 신규 |
| 사업계획 수립·관리 | 발주 예측 (수주 역산) | 🟡 로직O·데이터△ |
| 전사 보고 | 보고서 자동생성 (PPT) | 🟢 **PPT 우대요건 직격** |
| (리스크관리) | 파트너 건전성 모니터링 | 🟡 Fluence만 |

자격요건 매칭: *데이터 분석/처리* → 대시보드 전체 / *PPT* → Report Studio / *엑셀* → 별도 워치리스트 시트(아래 5절) / *비즈니스 영어* → 해외 RSS·SEC 영문 소스 처리.

## 2. 구현 완료 vs 로드맵 — 데모 경계선

면접 데모는 "많이"가 아니라 **버벅임 없이 돌아가는 것만**.

### A. 데모 핵심 (반드시 완벽하게) — "이건 진짜 됩니다"

| 화면 | 무엇 | 왜 핵심 | 안정성 |
|---|---|---|---|
| **파트너 인텔리전스** | SEC EDGAR로 Fluence 백로그·매출 실시간 | 공식 데이터 실시간 = 기술 신뢰도 | ★★★ 키 불필요 |
| **보고서 자동생성** | 버튼 → 6슬라이드 PPT 다운로드 | AX 임팩트 최고점 | ★★ OpenAI 키 |
| **시장분석(EIA)** | 미국 ESS 설치용량 차트 | "내 시장이 큰다"의 공식 근거 | ★★ |

### B. 보조 데모 — "이런 것도 됩니다"

뉴스 AI 선별(임베딩+GPT 2단계), 경쟁사 RSS 이벤트 분류. 실시간 RSS·OpenAI 의존이라 데모 중 느릴 수 있어 *설명용*으로만.

### C. 로드맵 (말로만 설명) — "다음은 이렇게"

> 적극성 + 비전을 가장 싸게 보여주는 층. 구현 안 해도 됨.

- **발주 예측 실데이터화** — 현재 수주 데이터가 하드코딩 → **DART 수주공시 + SEC 8-K 보도자료**로 자동 수집.
- **파트너 건전성 확장** — Fluence 1곳 → 공개 파트너 다수 (SEC=미국, DART=국내, FMP=기타).
- **데이터센터·신규시장** — World Bank·무역통계로 국가비교.
- **DB(Supabase) + 스케줄 자동화** — 이벤트 테이블 + 매일 무인 갱신.

## 3. 30초 데모 시나리오

`문제 → 추적 → 자동화 → 가치` 4박자, 화면 전환 3번.

> **[파트너 인텔리전스]** "에이스의 최대 약점은 매출이 Fluence·Powin 두 곳에 쏠린 고객 집중이었고, 2024년 Powin 파산으로 현실이 됐습니다. 그래서 남은 핵심 고객 Fluence의 수주잔고를 **SEC 공식 데이터로 실시간 추적**합니다 — 이 백로그가 에이스 미래 발주의 선행지표거든요.
> **[보고서 버튼 클릭 → PPT 다운로드]** 오늘의 시장 신호를 AI가 분석해 **버튼 하나로 경영진 보고용 PPT까지 자동 생성**합니다.
> **[마무리]** 신입이 매일 하던 시장 모니터링을 자동화한 도구입니다. 입사하면 바로 실무에 붙일 수 있습니다."

| 단계 | 화면 | 메시지 |
|---|---|---|
| 문제 | 파트너 인텔리전스 | 에이스의 진짜 고민(고객집중)을 안다 |
| 추적 | SEC 백로그 | 공식 데이터를 다룰 줄 안다 |
| 자동화 | 보고서 생성 | AX로 일을 줄인다 |
| 가치 | 마무리 | 입사 즉시 쓸 수 있다 |

## 4. 솔직한 한계 & 개선 로드맵 (회고력 어필)

면접에서 *한계를 먼저 말하는 것*이 신입의 가장 강한 신호.

- **발주 예측 탭이 현재 샘플/하드코딩 데이터** — 가장 큰 구멍. DART·SEC 실데이터화가 1순위 과제.
- **EIA는 '운영 중(후행)' 용량만 봄** — '계획·건설(planned)' 데이터로 바꾸면 *선행지표*가 됨.
- **데이터센터는 쓸만한 무료 실시간 API가 없음** — 공개 리포트 + 하이퍼스케일러 CapEx + 뉴스로 *조합 추정*. 한계를 안다고 명시하는 게 오히려 강점.
- **주가 정보는 사업기획에 거의 무용** — 백로그·수주공시 같은 사업 신호로 대체(이미 그렇게 설계함).
- **파이프라인 칸반은 영업 영역** — 기획 도구에선 축소/제외.

## 5. 엑셀 우대요건 대응 (별도)

JD의 "엑셀 활용 우수자"는 도구의 엑셀 출력 기능이 아니라 **본인의 실무 엑셀 손기술**(VLOOKUP·피벗·함수). 대응: **공개 파트너·경쟁사 워치리스트**를 엑셀 한 장으로 만들어 — VLOOKUP 조인 + 피벗 요약 + 위험 조건부서식 — 손기술을 직접 증명. (선택, 만들면 강력)
